Premium cinematic single-page site built with Next.js, Tailwind CSS, and GSAP.

## Run

```bash
npm install
npm run dev          # foreground, or:
bash scripts/start-localhost.sh   # background — prints URL, PID in .local/dev.pid
```

Stop the background server:

```bash
kill $(cat .local/dev.pid)
```

## Build

```bash
npm run build && npm start
```

## Optional: generated assets (Replicate)

1. Copy `.env.example` to `.env.local`
2. Add your token: `REPLICATE_API_TOKEN=r8_...` (never commit `.env.local`)
3. Run `npm run assets:generate`

The site works fully without this — deterministic fallbacks are used instead.

## Structure

- `src/app/` — App Router pages, layout, global styles (design tokens in `globals.css`)
- `src/components/sections/` — one component per page section
- `src/components/ui/BrandVideo.tsx` — the site's video component (poster,
  reduced-motion, mobile-safe)
- `public/assets/` — `generated/` (Replicate output), `fallbacks/` (deterministic),
  `video/` (the site's video asset(s) + posters)
- `scripts/` — localhost runner, secret scan, asset/video generation

## Video & sound

This site ships an active video experience — the hero video plays automatically
(muted) and/or a section is scroll-driven (see `SITE_BRIEF.md` → Video for the
PASS/FAIL grading, files, placement, and how to regenerate or upgrade).

If a soundtrack is included, it never autoplays: sound starts only from the
"Enable sound" / "Ativar trilha" control, defaults to off, and can be paused at any
time (see `SITE_BRIEF.md` → Music).

## Deploy

Vercel-ready: `npx vercel` or connect the repo. Set `REPLICATE_API_TOKEN` only if you
use the asset script in CI (not needed at runtime).
